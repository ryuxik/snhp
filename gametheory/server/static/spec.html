SNHP-NX/1 — package deals for agent payments

Agent-payment protocols are shaped like a checkout: the cart resolves to one price, and the buyer can pay or not. There is no field for trading quantity against delivery date. NX adds one — an open extension that mounts between the cart step and the payment step and hands back a settled package.

OUR PRE-REGISTERED KILL FIRED — READ THIS FIRST

We registered in advance that if price-only haggling reached 80% of NX's deal rate, the claim "you need bundling to close the deal" was dead. On 240 procurement scenarios it reached 94.2%. NX does not claim bundling is needed to close a deal.

WHAT SURVIVED

Packages capture more surplus. Share of the achievable joint surplus actually realized, same scenarios:

Tight deadlines are where bundling bites. In a high-urgency re-run, price-only recovers only 0.696 of NX's deal formation. That run is exploratory, not pre-registered, and says so when you run it.

WHAT IT DOES NOT DO

SCOPE, STATED UP FRONT
  • Moves no money, holds no goods. The host protocol settles the package; NX only gets it agreed.
  • Does not police fairness. It never verifies the seller's costs or the buyer's valuation.
  • Attestation is not settlement. It proves what was agreed and that nothing priced above list. It does not prove delivery.
  • Every measurement used scripted policies. They show what the message set can reach, not how clever an agent is.
  • Single seller, one line per item. Multi-seller packages and in-session re-quotes are open questions, not solved.

THE ACTUAL SPEC
