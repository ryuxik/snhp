SNHP Certificate Build Results

Your AI agent agreed to a price. Prove it was a good one.

Software is starting to spend real money. SNHP signs a receipt for every price an agent agrees to — what was listed, what was accepted, what it saved. Anyone can check it offline, without asking us. The negotiation engine is free. The proof is what we sell.

A log says we recorded that we behaved. A receipt says here is evidence, checkable without us. And a receipt claiming a price above the list price cannot be built at all — the code refuses to make one.

WHAT'S IN ONE

list_price      2670.00      the public list
quoted_price    2536.50      what was agreed
saving           133.50      list − quoted
c_eff           2100.00      the seller's floor

conditions      { a: true,  never above list
                  b: true,  floor didn't move with the buyer's story
                  c: true } disagreement point is the posted menu

context_hash      9f2c…      same inputs ⇒ same price
disclosure_digest 4b81…      digest only — never the raw willingness-to-pay
prev_hash         a07e…      chains to the previous receipt
key_source        "env"      or "ephemeral" — the signer is disclosed
notary_sig        ed25519…  breaks if any byte above changes

CHECK ONE WITHOUT TRUSTING US

python -m core.notary verify receipts.jsonl

Fetch the public key once from /v1/notary/key and you never need to contact us again — including years later, when the receipt matters and we might not be around.

WHAT IT DOES NOT PROVE

THE LIMITS, STATED UP FRONT
  • Only what happened inside the mechanism. A deal negotiated elsewhere is not covered.
  • Not that the buyer told the truth, unless an attestation token is attached. Without one, the receipt says so.
  • When scarcity legitimately moves the seller's floor, the receipt reports that and marks the condition false. We ship an honest false rather than a flattering true.
