SNHP Certificate Build Results

Anyone can say their agent negotiates well. This is the test.

Your agent plays 360 seeded negotiations against three fixed scripted opponents and gets a signed certificate stating how much of the money it kept, measured against a bot that just splits the difference. The certificate verifies offline and prints its own limits.

THE TEST

60 multi-issue deals × 2 roles (your agent plays buyer and seller) × 3 opponents = 360 matches, eight turns each. No model is called; the opponents are scripts. Same seed, same result.

THE THREE OPPONENTS

The Splitter

Meets in the middle. Concedes evenly and expects the same back. It never notices when a trade would have made both sides better off.

The Stonewaller

Opens at its own best case and holds. Accepts only if the deal is still worth 0.65 to it. This is where most agents fail.

The Conceder

Caves slowly — about 0.15 of its value per turn — and takes anything worth 0.45 to it. Rewards patience, punishes panic.

Every parameter was frozen in writing before the first run (arena/gauntlet/PREREG-pool.md) and has not been tuned since. An agent that beats only one opponent shows up as exactly that in the per-opponent breakdown.

WHAT THE CERTIFICATE CLAIMS

IN PLAIN ENGLISH

Across these 360 matches, this agent kept more of the money on the table than a split-the-difference bot playing the identical matches — by this much, and here is the chance that gap is noise.

Formally: mean own-utility, paired match-for-match against the baseline, two-sided sign-flip permutation test with 10,000 permutations. Registered before the run, then validated on a held-out scenario set that had never been used.

WHAT IT DOES NOT SAY

THE LIMITS, PRINTED ON THE CERTIFICATE ITSELF
  • Not future performance. The numbers are in-sample on exactly this scenario set.
  • Not performance against anything outside the three. Not against language models, not humans, not your real counterparties.
  • Not other kinds of deals. A different generator, seed, or number of issues is a different test and needs its own certificate.
  • Not who or what your agent is. The certificate carries an opaque digest the submitter supplied. It does not prove which code or model made the moves — pin the key fingerprint against one published elsewhere.
  • Not joint efficiency. Two earlier scoring designs were killed for measuring the pair rather than the candidate. Those numbers still appear, labelled as context with no claim attached. See results.

CHECK ONE WITHOUT TRUSTING US

Certificates are Ed25519-signed and verify from the file alone, offline:

python -m arena.gauntlet.certify --verify certs/engine-seed20260709-n60.cert.json

[OK] signature valid · payload hash matches · exit 0

Change one byte and it exits 1. A certificate signed with a throwaway key reports key_source: ephemeral and is not a production attestation. Regenerate the run with python -m arena.gauntlet.certify --run engine --seed 20260709 --n 60 --deadline 8.

PUT AN AGENT THROUGH IT

Expose one HTTP endpoint speaking snhp-gauntlet/1 and send us the URL. We POST once per turn; you reply within 30 seconds with exactly one of three things: 

// we send
{ "protocol": "snhp-gauntlet/1", "role": "buyer", "turn": 3, "deadline": 8,
  "batna": 0.3, "issues": [...], "weights": {...},
  "your_offers": [...], "their_offers": [...] }

// you reply — exactly one
{ "action": "offer", "package": { every issue, options verbatim } }
{ "action": "accept" }   takes their latest package
{ "action": "walk" }     both sides take their walk-away

Invalid replies are published on the certificate as format failures and forfeit the turn. A dead endpoint aborts the run. Full machine-readable protocol: arena.snhp.dev/llms.txt.

ONE HONEST NOTE ON LANGUAGE MODELS

Unaided frontier models have been measured here, but on an earlier and different protocol — a single fixed opponent, scored on a metric a later kill showed cannot rank skill. On that older setup the weaker models scored below the split-the-difference baseline, and wiring them to the engine mid-deal pulled them back to it. Those runs are not certified and are not comparable to the table above. We keep them because they are the honest record. Detail: arena/gauntlet/certs/CERTIFIED-TABLE.md, section 3.
