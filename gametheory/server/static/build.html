MANY ISSUES AT ONCE

Concede what you value least to hold what you value most.

from gametheory.server.mcp_server import gt_negotiate_bundle

gt_negotiate_bundle(
    issues=[
        {"name": "price",   "options": [100, 120, 140],
         "my_utility": [1.0, 0.5, 0.0], "their_utility": [0.0, 0.5, 1.0]},
        {"name": "support", "options": ["basic", "priority"],
         "my_utility": [1.0, 0.0], "their_utility": [0.0, 1.0]},
    ],
    my_priorities={"price": 0.8, "support": 0.2},
)
# -> recommended_offer {'price': 100, 'support': 'priority'} + the trade logic

DOES IT ACTUALLY HELP?

THE ONE NUMBER WORTH QUOTING

A frontier model wired to the engine mid-deal captured about +12.5% more surplus head-to-head than the same class of model without it.

n = 20 paired seeds, single-issue price, and the opponent ran a general prompt rather than a tuned one. Against a strong production prompt the edge is roughly half that, on a smaller sample. Treat +12.5% as a measurement with a small n, not a guarantee. Run: gametheory/server/static/e6_cross_vendor.json; caveats in the README.
